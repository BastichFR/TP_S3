Introduction

The purpose of this practical is to give you some basic knowledge about multi-process programming. You are going to learn how to create new processes from a running process and how these processes can communicate with each other.

For the whole practical, you are not allowed to include any other files than those already included.

The fork.c file

fork.c
#include <err.h>
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>

#define PRINT(fmt_, ...) printf("%i: " fmt_ "\n", getpid() __VA_OPT__(,) __VA_ARGS__);

int main()
{
    // Initializes x.
    int  x = 0;

    // Forks the process.
    // (Creates a new process.)
    int id = fork();

    // Exits if error.
    if (id == -1)
        err(EXIT_FAILURE, "fork()");

    // Parent process.
    else if (id)
    {
        PRINT("Inside parent");
        PRINT("Child id : %i", id);
        PRINT("Initial x: %i", x);
        x = -42;
    }

    // Child process.
    else
    {
        PRINT("Inside child");
        PRINT("Initial x: %i", x);
        x = 42;
    }

    // Parent and child processes.
    PRINT("Parent id: %i", getppid());
    PRINT("New x: %i", x);

    return EXIT_SUCCESS;
}

The sort.c file

sort.c
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>
#include <err.h>
#include <sys/wait.h>
#include <time.h>

#define PRINT(fmt_, ...) printf("%u: " fmt_ "\n", getpid() __VA_OPT__(,) __VA_ARGS__);

void random_fill(int tab[], size_t len)
{
    for (int *cur = tab; cur != tab + len; ++cur)
        *cur = random() % 1024;
}

void swap(int* a, int* b)
{
    int c = *a;
    *a = *b;
    *b = c;
}

const char* is_sorted(int tab[], size_t len)
{
    size_t j = 1;

    while (j < len && tab[j - 1] <= tab[j])
        j++;

    return j == len ? "Yes" : "No";
}

void sort1(int tab[], size_t len)
{
    for (size_t i = 0; i < len; ++i)
    {
        size_t min = i;

        for (size_t j = i + 1; j < len; ++j)
            if (tab[j] < tab[min])
                min = j;

        swap(tab + i, tab + min);
    }
}

void sort2(int tab[], size_t len)
{
    int move = 1;

    for (size_t i = 0; i < len && move; ++i)
    {
        move = 0;

        for (size_t j = 0; j < len - i - 1; ++j)
        {
            if (tab[j] > tab[j + 1])
            {
                move = 1;
                swap(tab + j, tab + j + 1);
            }
        }
    }
}

double diff_timespec(const struct timespec* t1, const struct timespec* t0)
{
    double second = difftime(t1->tv_sec, t0->tv_sec);
    return second + ((double)t1->tv_nsec - (double)t0->tv_nsec) / 1e9;
}

int main(int argc, char** argv)
{
    // If the number of arguments is not valid,
    // exits with an error message.
    if (argc != 2)
        errx(EXIT_FAILURE, "Usage: %s ARRAY_SIZE (from 1 to 100000)", argv[0]);

    // Gets the size of the array.
    size_t size = strtoul(argv[1], NULL, 10);
    if (size == 0 || size > 100000)
        errx(EXIT_FAILURE, "The argument is not valid.");

    // Declares an array.
    int a[size];

    // Fills the array with random values.
    random_fill(a, size);
    PRINT("Sorted: %s", is_sorted(a, size));

    // TODO

    return EXIT_SUCCESS;
}

Simple Call to fork()

The first step is to understand the fork.c file that contains a simple call to fork(). So, compile it and run it several times.

Note that the PRINT() macro is used to print a string of characters with the same syntax as printf(), but with the following two differences:

  • It prefixes the string with the process identifier.
  • It appends a newline character to the string.

Sorting Arrays

You have to complete the sort.c file in order to time two different sorting algorithms.

Two functions are given: sort1() and sort2(). You will call these two functions separately in two child processes and time the two children from the parent.

When you execute the program, you have to specify the size of an array as a parameter. The array must be large enough so that it takes some time to be sorted. At the beginning of the program, the array is filled with random values.

You should follow the following steps:

  • Get the current time (t0) with the clock_gettime(2) function. You will have to use the timespec structure.
  • Create a new first child process that:
    • Sorts the array with sort1().
    • Displays the return value of is_sorted() in order to be sure that the array is sorted (use the PRINT() macro).
    • Returns EXIT_SUCCESS.
  • Wait for the end of the child process by using wait(2).
  • Get the current time (t1).
  • Print the difference between t1 and t0. You can use the given diff_timespec() function.
  • Repeat the previous step to create a second child process that calls sort2().

The chg_case.c File

chg_case.c
#include <stdlib.h>
#include <unistd.h>
#include <string.h>
#include <err.h>

const size_t BUFFER_SIZE = 32;

void to_lower(int in_fd, int out_fd)
{
    char buffer[BUFFER_SIZE];
    ssize_t r;

    while ((r = read(in_fd, buffer, BUFFER_SIZE)))
    {
        if (r == -1)
            err(EXIT_FAILURE, "Could not read data");

        for (size_t i = 0; i < (size_t)r; i++)
            if (buffer[i] >= 'A' && buffer[i] <= 'Z')
                buffer[i] += 'a' - 'A';

        if (write(out_fd, buffer, r) != r)
            errx(EXIT_FAILURE, "Could not write data");
    }
}

void to_upper(int in_fd, int out_fd)
{
    char buffer[BUFFER_SIZE];
    ssize_t r;

    while ((r = read(in_fd, buffer, BUFFER_SIZE)))
    {
        if (r == -1)
            err(EXIT_FAILURE, "Could not read data");

        for (size_t i = 0; i < (size_t)r; i++)
            if (buffer[i] >= 'a' && buffer[i] <= 'z')
                buffer[i] += 'A' - 'a';

        if (write(out_fd, buffer, r) != r)
            errx(EXIT_FAILURE, "Could not write data");
    }
}

int main(int argc, char** argv)
{
    if (argc == 1)
        to_lower(STDIN_FILENO, STDOUT_FILENO);

    else if (argc == 2)
    {
        if (strcmp(argv[1], "-l") == 0)
            to_lower(STDIN_FILENO, STDOUT_FILENO);
        else if (strcmp(argv[1], "-U") == 0)
            to_upper(STDIN_FILENO, STDOUT_FILENO);
        else
            errx(EXIT_FAILURE, "Usage: %s [-l|-U]", argv[0]);
    }

    else
        errx(EXIT_FAILURE, "Usage: %s [-l|-U]", argv[0]);

    return EXIT_SUCCESS;
}

Introduction

In this section, you will write a program (the manager) that sends the contents of a text file to another program (the worker) and gets a response from the worker.

Implementation

The Worker

The worker can be any program that reads the standard input, processes the data and sends the response to the standard output. In order to simplify the manager code, the worker must not change the number of bytes, that is, the number of bytes sent to the standard output must be the same as the number of bytes read from the standard input. The worker can only change the values of the bytes (not the number of bytes).

The code of a worker is given: chg_case.c. You are not allowed to modify this file. Look at it carefully in order to understand it.

The chg_case program can convert lowercase letters to uppercase, and vice versa. Two options can be used:

  • The -l option, which converts letters to lowercase (the default option).
  • The -U option, which converts letters to uppercase.

Compile the file (with the given Makefile) and use the following examples to test it. 

$ make chg_case

$ echo Hello World | ./chg_case -U

The Manager

The manager can take a variable number of arguments:

  • The first argument is a text file.
  • The second argument is the worker (an executable file).
  • The other arguments are those passed to the worker.

The behavior of the manager can be summarized by the following diagram:

The manager must create two pipes and fork the process.

The child process has to:

  • Redirect some sides of the pipes (and close the others).
  • Execute the worker.

The parent process has to:

  • Close the unused sides of the pipes.
  • Send the contents of the text file to the worker (via pipe 1).
  • Read the response from the worker (via pipe 2).
  • Send the response to stdout.
  • Close the file and the pipes.

As the size of the text file cannot be known at compile time, you have to use a buffer to send the contents of the file and to read the response.

To read the text file, you first need to open it with the open(2) function. Use the O_RDONLY flag. That function returns a file descriptor that can be used with read(2).

To simplify the error management, we consider that the write(2) function must return a value equal to the third parameter. If not, you can exit by calling the errx(3) function. For instance: 

if (write(STDOUT_FILENO, buffer, size) != size)
    errx(EXIT_FAILURE, "Could not write data to stdout");
